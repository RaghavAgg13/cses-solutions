## 📁 File Naming Convention
Files follow the CSES problem naming scheme:
```
<problem_id>.<extension>
```
**Examples:**
- `1068.py` → CSES Problem 1068
- `1085.cpp` → CSES Problem 1085 (solved in C++)
- `1097.c` → CSES Problem 1097 (solved in C)

**Special suffixes you might see:**
| Suffix | Meaning |
|--------|---------|
| `- fail` | Incorrect solution — kept for reference |
| `- tle` | Time Limit Exceeded — kept for reference |
| `- wa` | Wrong Answer attempt |
| `- slow` | Brute-force or suboptimal solution — kept for reference |
| `- n^2` | Quadratic solution — kept for reference |
| `- wtf` | Particularly tricky or counterintuitive problem |
| `- pq stl` | Alternative implementation using STL priority queue |

## ▶️ Running a Solution
### Python
```bash
python3 1068.py
```
### C++
```bash

./a
(./a.out on Linux)
```
### C
```bash
gcc 1097.c
./a
```

## 📌 Notes
- Files with `- fail`, `- tle`, or `- slow` suffixes are intentionally kept — they're useful for understanding what *doesn't* work.
- Some problems have both a slower and optimised version (e.g. `2163 - slow.cpp` and `2163.cpp`) — check both for the progression of the approach.
